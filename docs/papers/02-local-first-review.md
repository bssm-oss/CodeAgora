# 로컬 우선 코드 리뷰 도구의 설계

## 초록
본 논문은 CodeAgora의 로컬 우선(Local-first) 코드 리뷰 도구 설계와 구현 원리를 다룬다. 기존의 코드 리뷰 자동화 도구는 중앙 집중형 서버 또는 SaaS 환경에 의존하는 경우가 많으나, CodeAgora는 CLI, 로컬 설정, 세션 저장소, 선택적 GitHub Actions 통합을 통해 사용자의 로컬 환경에서 코드 리뷰를 수행할 수 있도록 설계되었다. 본 논문은 로컬 우선 접근법의 장점과 한계, 시스템 구조, 보안 및 프라이버시 고려사항, 실제 적용 시나리오와 평가 계획을 상세히 기술한다. 또한, 중앙 서버 기반 도구와의 비교를 통해 로컬 우선 설계의 실질적 trade-off를 논의한다.

## 키워드
로컬 우선, 코드 리뷰, 자동화, CLI, 프라이버시, GitHub Actions, 세션 저장소

## 1. 서론
코드 리뷰 자동화 도구의 발전에도 불구하고, 많은 개발자들은 여전히 프라이버시, 인증 정보 관리, 네트워크 의존성, 환경 맞춤화 등 다양한 이유로 로컬 환경에서의 리뷰를 선호한다. CodeAgora는 이러한 요구를 반영하여, 사용자의 로컬 환경에서 코드 리뷰 파이프라인을 실행할 수 있도록 설계된 도구이다. 본 논문은 로컬 우선 코드 리뷰 도구의 설계 원리와 구현 방식을 체계적으로 정리하고, 실제 개발 및 운영 과정에서의 경험을 공유한다.

## 2. 배경 및 문제 정의
기존의 코드 리뷰 자동화 도구는 대부분 중앙 서버 또는 SaaS 환경에서 동작하며, 사용자는 코드를 서버에 업로드하거나 외부 인증 정보를 제공해야 한다. 이로 인해 프라이버시 침해, 인증 정보 유출, 네트워크 장애 시 사용 불가, 환경 맞춤화의 어려움 등 다양한 문제가 발생한다. 특히, 보안이 중요한 프로젝트나 사내망 환경에서는 중앙 서버 기반 도구의 도입이 제한적일 수 있다. 본 연구는 이러한 한계를 극복하기 위해, 로컬 우선(Local-first) 설계 원칙을 도입하였다.

## 3. 시스템 설계
CodeAgora의 로컬 우선 코드 리뷰 도구는 다음과 같은 구조로 설계되었다.

1. **로컬 설정 및 인증 정보 관리**: `.ca/config.json` 파일과 로컬 credential 저장소를 통해, 사용자는 자신의 환경에 맞는 설정과 인증 정보를 안전하게 관리할 수 있다.
2. **세션 및 결과 저장**: 리뷰 세션과 결과는 로컬 파일 시스템 내에 저장되며, 사용자는 언제든지 이전 리뷰 내역을 조회하거나 복원할 수 있다.
3. **선택적 GitHub Actions 통합**: 필요에 따라 GitHub Actions 또는 MCP(Multi-Context Protocol)와 연동하여, 협업 환경에서도 자동화 리뷰를 확장할 수 있다.
4. **CLI 및 local UI 지원**: CLI 기반 인터페이스, MCP, GitHub Actions, 그리고 desktop local UI를 통해 사용자는 자신의 작업 방식에 맞게 도구를 활용할 수 있다.

이러한 구조는 사용자의 프라이버시와 보안을 보장하면서도, 필요 시 중앙 서버와의 연동을 통해 확장성을 확보한다.

## 4. 평가 계획 및 예비 근거
- 로컬 credential permission 검증 및 보안성 평가
- 중앙 서버 기반 도구 대비 데이터 이동 경로, 프라이버시 보호 수준 비교
- 온보딩 및 환경 설정 실패 유형 분석
- 실제 PR 리뷰 시나리오에서의 latency, 사용성, 오류 발생률 측정
- 예비 테스트는 `src/tests/config-*` 및 실제 로컬 환경에서 진행 중이며, 추가적인 사용자 연구가 필요하다.

## 5. 논의 및 한계
- 로컬 설정의 복잡도가 사용자의 숙련도에 따라 달라질 수 있다.
- 프라이버시와 보안은 강화되나, 협업 및 중앙 관리 기능은 상대적으로 약할 수 있다.
- 프라이버시 보호와 협업 효율성 간의 trade-off가 존재한다.
- threat model, `.ca/` 데이터 흐름 다이어그램, 서버 기반 도구와의 비교표 등 추가 자료가 필요하다.

## 6. 결론
CodeAgora의 로컬 우선 코드 리뷰 도구는 프라이버시와 보안을 중시하는 개발 환경에서 효과적으로 활용될 수 있다. 로컬 설정, 세션 저장, 선택적 중앙 연동 구조를 통해 다양한 사용자의 요구를 충족하며, 향후 실증적 평가와 사용자 연구를 통해 효과성을 검증할 예정이다.

## 확장 본문 초안

코드 리뷰 자동화는 편리하지만, 소스 코드와 인증 정보를 외부 서비스로 전달한다는 부담을 동반한다. 기업이나 오픈소스 maintainer는 리뷰 자동화의 이점을 원하면서도, 코드 유출, credential 관리, vendor lock-in, 서버 장애 의존성을 우려한다. CodeAgora의 로컬 우선 설계는 이런 우려를 줄이기 위해 CLI와 로컬 설정을 기본 실행 경로로 삼는다.

로컬 우선이라는 말은 모든 처리가 오프라인에서 끝난다는 뜻은 아니다. LLM provider를 호출하려면 외부 API나 CLI 도구를 사용할 수 있다. 중요한 차이는 orchestration, configuration, session storage, review artifact의 통제권이 사용자의 프로젝트와 로컬 환경에 남는다는 점이다. GitHub Actions, MCP, 그리고 desktop app은 이 로컬 파이프라인을 확장하는 선택지다.

서버 중심 코드 리뷰 도구는 설정이 단순하고 중앙에서 경험을 통제할 수 있다. 그러나 모든 코드 변경이 외부 서비스로 이동하고, 결과 저장과 접근 제어도 서비스 정책에 의존한다. 반면 로컬 도구는 설치와 설정 부담이 있지만, 코드와 결과의 위치를 사용자가 명확히 알 수 있다. CodeAgora는 `.ca/config.json`, 로컬 credential, session directory를 통해 이 경계를 명시한다.

로컬 우선 설계의 또 다른 장점은 재현성이다. 동일한 diff, config, provider 상태를 기록하면 이후 session을 다시 설명하거나 비교할 수 있다. 이는 explainable review session과도 연결된다. CLI에서 생성한 session은 desktop app이나 MCP를 통해 다시 탐색될 수 있으며, 이 과정에서 중앙 서버에 session을 업로드할 필요가 없다.

평가는 threat model로 시작해야 한다. 어떤 데이터가 로컬에 남고, 어떤 데이터가 provider로 전송되는지, credential은 어디에 저장되는지 명시해야 한다. 또한 onboarding 평가에서는 설치 실패, credential permission 문제, config 오류를 측정할 수 있다. 로컬 우선은 만능이 아니지만, 민감한 코드 리뷰 workflow에서는 통제권과 투명성이라는 명확한 장점을 제공한다.
