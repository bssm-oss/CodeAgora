모든 도메인을 동일한 5개 기준으로 채점한다. 각 기준의 점수는 비율(0~100%)로 매긴 뒤 가중치를 곱해 합산한다.

| 기준 | 가중치 | 측정 방법 |
|------|--------|----------|
| **A. 기능 구현** | 40% | 명세(CLAUDE.md/PRD) 대비 구현된 모듈·함수 비율. 스텁은 0점, 부분 구현은 비례 배점 |
| **B. 테스트** | 20% | 해당 도메인 전용 테스트 파일 유무 × 테스트 수 × 통과율. 0건이면 0점 |
| **C. 에러/엣지 처리** | 15% | fallback, timeout, graceful degradation, retry 존재 여부. 없으면 0점 |
| **D. 문서-코드 일치** | 10% | CLAUDE.md/README 명세와 실제 구현의 괴리. 괴리 없으면 만점 |
| **E. 코드 청결도** | 15% | 미사용 export, `any`, TODO/FIXME, 불필요 import. 0건이면 만점 |
