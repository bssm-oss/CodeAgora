파일 경로와 코드 인용은 확인됨. 프리셋 별칭과 CLI 백엔드 제외 정책은 사람 검토로 올립니다.

단일 프롬프트가 아니라 리뷰 시스템

모델 의견을 그대로 믿지 않고, 검증 가능한 결론으로 좁힙니다.

CodeAgora는 리뷰어 선택, 병렬 검토, hallucination 필터, 토론, 최종 판정을 분리해 약한 신호가 릴리스 결정을 흔들지 않게 만듭니다.

01

분석

변경 범위와 위험도를 먼저 계산해 리뷰어에게 필요한 맥락을 제공합니다.

02

병렬 검토

보안, 안정성, 유지보수성 관점의 리뷰어가 독립적으로 같은 diff를 봅니다.

03

토론

충돌된 지적은 supporter와 devil's advocate를 거쳐 근거를 다시 확인합니다.

04

판정

`ACCEPT`, `REJECT`, `NEEDS_HUMAN`과 함께 재검토 가능한 증거를 남깁니다.

공식 지원 표면

CLI, GitHub Action, MCP, Desktop이 같은 리뷰 의미론을 공유합니다.

CLI

터미널에서 diff 리뷰를 실행하고 세션 산출물을 남깁니다.

GitHub Action

PR에서 권한, secret, provider 실패를 안전하게 처리하며 리뷰 결과를 게시합니다.

MCP

IDE와 에이전트가 리뷰, 설정, 세션 도구를 같은 계약으로 호출합니다.

Desktop

로컬 세션 탐색, 리뷰 실행, 결과 확인, 증거 내보내기를 앱에서 다룹니다.

보안 경계

중개 프록시 없이, 자격 증명은 로컬에서 관리합니다.

CodeAgora는 사용자가 설정한 모델 공급자 API와 직접 통신합니다. API 키는 로컬 자격 증명 경로에 저장하고, 로그와 evidence에서 비밀값이 새지 않도록 redaction 경계를 둡니다.

credentials ~/.config/codeagora/credentials

`agora env set <provider>`로 저장되는 키는 로컬 파일 권한 경계 안에서 관리합니다.

provider direct OpenAI · Anthropic · OpenRouter

코드 리뷰 요청은 중간 CodeAgora SaaS 서버가 아니라 사용자가 지정한 공급자 엔드포인트로 향합니다.

fork safe Secret이 없는 PR은 안전하게 강하

GitHub Action 표면은 fork PR, 누락된 secret, provider 실패 같은 경로를 배포 계약 안에서 다룹니다.

검증 가능한 품질

마케팅 수치 대신, 로컬에서 돌릴 수 있는 테스트와 벤치마크를 둡니다.

저장소의 `benchmarks/`, 중앙 Vitest 스위트, release evidence 스크립트로 파이프라인 품질과 배포 주장을 검증합니다. 랜딩도 같은 원칙을 따라 확인 가능한 사실만 말합니다.

pnpm test pnpm --filter @codeagora/site smoke pnpm evidence:manifest -- --require=rc
