Head Verdict

파일 경로와 코드 인용은 확인됨. 프리셋 별칭과 CLI 백엔드 제외 정책은 사람 검토로 올립니다.

왜 필요한가

AI 리뷰를 더 많이 붙이는 것보다, 믿을 수 없는 지적을 걸러내는 일이 중요합니다.

비개발자에게도 핵심은 단순합니다. CodeAgora는 “이 코드를 합쳐도 되는가?”라는 질문에 답하기 위해, 주장마다 근거를 붙이고 약한 추측은 따로 분리합니다.

01

그럴듯한 오답을 줄입니다

AI는 때로 없는 파일, 틀린 라인, 실제 코드와 다른 인용을 말합니다. CodeAgora는 먼저 그런 주장을 확인해 제거하거나 낮은 신뢰도로 표시합니다.

02

서로 다른 관점을 한곳에 모읍니다

보안, 안정성, 유지보수성, 배포 영향은 서로 다른 문제입니다. 여러 리뷰어가 같은 변경을 각자 보고, 겹치는 지적과 충돌하는 지적을 분리합니다.

03

결정 피로를 낮춥니다

긴 코멘트 목록만 받으면 결국 사람이 다시 처음부터 읽어야 합니다. CodeAgora는 수정해야 할 항목, 나중에 봐도 되는 항목, 사람이 확인해야 할 질문을 나눕니다.

단일 프롬프트가 아니라 리뷰 시스템

CodeAgora는 리뷰 코멘트가 아니라, 결정 자료를 만듭니다.

코드 변경을 설명하고, 독립 리뷰를 모으고, 근거가 약한 주장을 걷어낸 뒤, 마지막에는 `ACCEPT`, `REJECT`, `NEEDS_HUMAN` 중 하나로 정리합니다. 개발자가 아닌 PM, 운영자, 팀 리더도 “무엇을 확인해야 하는지”를 읽을 수 있게 만드는 흐름입니다.

01

변경 내용 정리

먼저 어떤 파일이 바뀌었고, 설정·보안·배포 중 어디에 영향을 줄 수 있는지 정리합니다. 리뷰어가 추측으로 시작하지 않게 기본 맥락을 맞춥니다.

02

독립 검토

보안, 안정성, 유지보수성 관점의 리뷰어가 서로의 답을 보지 않고 같은 diff를 봅니다. 한 모델의 성향이 전체 결론을 끌고 가지 않게 합니다.

03

근거 확인

지적한 파일이 실제로 있는지, 라인 범위가 맞는지, 코드 인용이 조작되지 않았는지 확인합니다. 틀린 주장은 제거하고 애매한 주장은 낮은 신뢰도로 보냅니다.

04

토론과 반박

리뷰어들이 서로 다른 결론을 내면 supporter와 반대 관점이 다시 따집니다. 근거가 충분하면 결론을 좁히고, 부족하면 사람 확인으로 올립니다.

05

판정표

마지막 결과는 승인 가능, 수정 필요, 사람 확인 필요로 읽히는 판정표가 됩니다. 함께 남는 근거 덕분에 나중에 같은 결정을 다시 검토할 수 있습니다.

공식 지원 표면

어디서 실행해도 같은 기준으로 판단합니다.

터미널에서 직접 돌리든, GitHub PR에 붙이든, IDE 에이전트가 호출하든, Desktop 앱에서 보든 결과의 의미는 같습니다. 표면만 다르고 핵심 리뷰 파이프라인은 하나입니다.

CLI

개발자가 로컬에서 작업 중인 변경을 바로 점검할 때 씁니다. 터미널에서 diff를 넣으면 같은 리뷰 파이프라인을 실행하고, 나중에 확인할 세션 산출물을 남깁니다.

GitHub Action

팀이 Pull Request를 열 때 자동으로 실행됩니다. 권한, secret 누락, 외부 기여자의 fork PR처럼 조심해야 하는 상황은 무리하게 실패시키지 않고 상태를 분리해 보여줍니다.

MCP

AI IDE나 코딩 에이전트가 CodeAgora를 도구처럼 부를 수 있는 경로입니다. 에이전트가 임의로 해석하지 않고, 리뷰·설정·세션 도구를 정해진 계약으로 호출합니다.

Desktop

터미널이 익숙하지 않은 팀원도 로컬 세션, 리뷰 실행, 결과 확인, 증거 내보내기를 화면에서 다룰 수 있게 하는 앱 표면입니다. 별도 의미론을 만들지 않고 core 계약을 보여줍니다.

보안 경계

중개 서버 없이, 자격 증명은 로컬에서 관리합니다.

CodeAgora는 별도 SaaS 서버에 코드를 쌓아두는 구조가 아닙니다. 사용자가 설정한 모델 공급자 API와 직접 통신하고, API 키는 로컬 자격 증명 경로에 저장하며, 로그와 evidence에서 비밀값이 새지 않도록 redaction 경계를 둡니다.

credentials 키는 내 컴퓨터에 저장

`agora env set <provider>`로 저장되는 키는 `~/.config/codeagora/credentials` 아래에서 로컬 파일 권한 경계 안에 둡니다.

provider direct 모델 공급자로 직접 요청

리뷰 요청은 중간 CodeAgora SaaS 서버가 아니라 사용자가 지정한 OpenAI, Anthropic, OpenRouter 같은 공급자 엔드포인트로 향합니다.

fork safe 위험한 PR 경로는 분리

GitHub Action 표면은 fork PR, 누락된 secret, provider 실패 같은 경로를 한 덩어리 오류로 숨기지 않고, 안전하게 낮춘 상태로 보고합니다.

검증 가능한 품질

우리가 한 말을 테스트로 다시 확인합니다.

좋은 리뷰 도구라고 말하려면 실제로 틀린 지적을 줄이고, 중요한 문제를 놓치지 않고, 배포 표면이 같은 의미로 동작해야 합니다. 저장소의 `benchmarks/`, 중앙 Vitest 스위트, release evidence 스크립트는 그 약속을 반복해서 확인하는 장치입니다.
