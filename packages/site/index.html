기존 AI 리뷰와 무엇이 다른가요?

AI 리뷰를 많이 붙이는 것보다, 잘못된 지적을 차단하는 것이 핵심입니다.

비개발자에게도 핵심은 단순합니다. CodeAgora는 “이 코드를 합쳐도 되는가?”라는 질문에 답하기 위해, 주장마다 근거를 붙이고 약한 추측은 따로 분리합니다.

01

그럴듯한 오답을 줄입니다

AI는 때로 없는 파일, 틀린 라인, 실제 코드와 다른 인용을 말합니다. CodeAgora는 먼저 그런 주장을 확인해 제거하거나 낮은 신뢰도로 표시합니다.

02

서로 다른 관점을 한곳에 모읍니다

보안, 안정성, 유지보수성, 배포 영향은 서로 다른 문제입니다. 여러 리뷰어가 같은 변경을 각자 보고, 겹치는 지적과 충돌하는 지적을 분리합니다.

03

결정 피로를 낮춥니다

긴 코멘트 목록만 받으면 결국 사람이 다시 처음부터 읽어야 합니다. CodeAgora는 수정해야 할 항목, 나중에 봐도 되는 항목, 사람이 확인해야 할 질문을 나눕니다.

체계적인 자동 검증 프로세스

CodeAgora는 리뷰 코멘트가 아니라, 결정 자료를 만듭니다.

코드 변경을 설명하고, 독립 리뷰를 모으고, 근거가 약한 주장을 걷어낸 뒤, 마지막에는 `ACCEPT`, `REJECT`, `NEEDS_HUMAN` 중 하나로 정리합니다. 개발자가 아닌 PM, 운영자, 팀 리더도 “무엇을 확인해야 하는지”를 읽을 수 있게 만드는 흐름입니다.

01

변경 내용 정리

먼저 어떤 파일이 바뀌었고, 설정·보안·배포 중 어디에 영향을 줄 수 있는지 정리합니다. 리뷰어가 추측으로 시작하지 않게 기본 맥락을 맞춥니다.

02

독립 검토

보안, 안정성, 유지보수성 관점의 리뷰어가 서로의 답을 보지 않고 같은 diff를 봅니다. 한 모델의 성향이 전체 결론을 끌고 가지 않게 합니다.

03

근거 확인

지적한 파일이 실제로 있는지, 라인 범위가 맞는지, 코드 인용이 조작되지 않았는지 확인합니다. 틀린 주장은 제거하고 애매한 주장은 낮은 신뢰도로 보냅니다.

04

토론과 반박

리뷰어들이 서로 다른 결론을 내면 찬성하는 AI와 반대하는 AI가 서로의 논리를 다시 검증합니다. 근거가 충분하면 결론을 좁히고, 부족하면 사람 확인으로 올립니다.

05

판정표

마지막 결과는 승인 가능, 수정 필요, 사람 확인 필요로 읽히는 판정표가 됩니다. 함께 남는 근거 덕분에 나중에 같은 결정을 다시 검토할 수 있습니다.

공식 지원 환경

어디서 실행해도 같은 기준으로 판단합니다.

터미널에서 직접 돌리든, GitHub PR에 붙이든, IDE 에이전트가 호출하든, Desktop 앱에서 보든 결과의 의미는 같습니다. 실행 환경만 다르고 핵심 리뷰 파이프라인은 하나입니다.

CLI

개발자가 로컬에서 작업 중인 변경을 바로 점검할 때 씁니다. 터미널에서 diff를 넣으면 같은 리뷰 파이프라인을 실행하고, 나중에 확인할 세션 산출물을 남깁니다.

GitHub Action

팀이 Pull Request를 열 때 자동으로 실행됩니다. 권한, secret 누락, 외부 기여자의 fork PR처럼 조심해야 하는 상황은 무리하게 실패시키지 않고 상태를 분리해 보여줍니다.

MCP

AI IDE나 코딩 에이전트가 CodeAgora를 도구처럼 부를 수 있는 경로입니다. 에이전트가 임의로 해석하지 않고, 리뷰·설정·세션 도구를 정해진 계약으로 호출합니다.

Desktop

터미널이 익숙하지 않은 팀원도 로컬 세션, 리뷰 실행, 결과 확인, 증거 내보내기를 화면에서 다룰 수 있게 하는 RC 앱입니다. 별도 판단 규칙을 만들지 않고 같은 리뷰 결과를 보여줍니다.

어떤 검색어로 찾아도 같은 제품

AI 코드 리뷰 도구가 필요한 순간을 CodeAgora스럽게 정리했습니다.

CodeAgora는 단순히 “AI가 리뷰 코멘트를 써주는 도구”가 아닙니다. 로컬 diff, GitHub Pull Request, MCP 기반 AI IDE, Desktop RC까지 같은 판단 기준으로 연결해 팀이 결정할 수 있는 근거를 남깁니다.

AI 코드 리뷰 도구

한 번의 답보다 여러 관점의 합의를 보고 싶을 때

보안, 안정성, 유지보수성 리뷰어가 같은 변경을 따로 보고 충돌하는 주장을 다시 토론합니다. 최종 결과는 긴 감상문이 아니라 사람이 결정할 수 있는 판정표로 남습니다.

GitHub Action 코드 리뷰 자동화

Pull Request마다 같은 기준을 자동으로 붙이고 싶을 때

팀 저장소에서는 GitHub Action으로 PR 리뷰를 실행합니다. provider secret이 없거나 fork PR처럼 조심해야 하는 상황도 리뷰 실패와 보안 상태를 구분해서 다룹니다.

MCP 코드 리뷰

AI IDE와 코딩 에이전트가 리뷰 도구를 직접 부르게 할 때

MCP 서버는 에이전트가 CodeAgora의 리뷰, 설정, 세션 조회 기능을 정해진 도구 계약으로 호출하게 합니다. 에이전트마다 따로 리뷰 의미를 재해석하지 않아도 됩니다.

CLI diff review

로컬에서 커밋 전 변경을 빠르게 확인하고 싶을 때

터미널에서 현재 diff를 넘기면 동일한 멀티 LLM 리뷰 파이프라인이 실행됩니다. 로컬 세션 산출물이 남기 때문에 나중에 같은 판단을 다시 확인할 수 있습니다.

Desktop RC

터미널이 익숙하지 않은 팀원도 결과를 읽어야 할 때

Desktop은 로컬 리뷰 세션과 결과 확인을 화면에서 다루기 위한 RC 앱입니다. 별도의 판단 규칙을 만들지 않고 CLI, Action, MCP와 같은 결과 의미를 보여주는 방향입니다.

보안과 프라이버시

중개 서버 없이, API 키는 내 컴퓨터에서 관리합니다.

CodeAgora는 별도 SaaS 서버에 코드를 쌓아두는 구조가 아닙니다. 사용자가 설정한 모델 공급자 API와 직접 통신하고, API 키는 로컬 파일 권한 안에 저장하며, 로그와 검토 기록에서는 비밀값을 가려서 남깁니다.

credentials 키는 내 컴퓨터에 저장

`agora env set <provider>`로 저장되는 키는 `~/.config/codeagora/credentials` 아래에서 로컬 파일 권한 경계 안에 둡니다.

provider direct 모델 공급자로 직접 요청

리뷰 요청은 중간 CodeAgora SaaS 서버가 아니라 사용자가 지정한 OpenAI, Anthropic, OpenRouter 같은 공급자 엔드포인트로 향합니다.

fork safe 위험한 PR 경로는 분리

GitHub Action 연동은 fork PR, 누락된 secret, provider 실패 같은 경로를 한 덩어리 오류로 숨기지 않고, 안전하게 낮춘 상태로 보고합니다.

자주 묻는 질문

검색해서 들어온 사람이 바로 확인해야 할 답입니다.

CodeAgora의 핵심은 “AI가 코멘트를 많이 다는 것”이 아니라, 리뷰 주장을 검증하고 사람이 결정할 수 있게 만드는 것입니다. 아래 답변은 현재 RC 채널 기준의 공개 약속만 담습니다.

CodeAgora는 무엇인가요?

CodeAgora는 여러 AI 리뷰어가 같은 코드 변경을 독립적으로 검토하고, 파일·라인·코드 인용 같은 근거를 확인한 뒤, ACCEPT, REJECT, NEEDS_HUMAN 판정으로 정리하는 AI 코드 리뷰 도구입니다.

하나의 AI 코드 리뷰와 무엇이 다른가요?

한 모델의 답을 그대로 믿지 않습니다. 여러 리뷰어가 다른 관점으로 같은 diff를 보고, 틀린 파일이나 조작된 인용처럼 그럴듯한 오답을 걸러내며, 충돌하는 주장은 토론과 반박으로 좁힙니다.

GitHub Action 코드 리뷰 자동화에 쓸 수 있나요?

네. GitHub Action으로 Pull Request마다 CodeAgora 리뷰를 실행할 수 있습니다. fork PR, 누락된 secret, provider 실패처럼 주의가 필요한 경로는 한 덩어리 오류로 숨기지 않고 안전한 상태로 분리해 보여주는 방향입니다.

MCP는 왜 필요한가요?

MCP는 AI IDE와 코딩 에이전트가 CodeAgora를 도구처럼 호출하게 하는 연결 방식입니다. 리뷰 실행, 설정 확인, 세션 조회를 정해진 계약으로 다뤄서 에이전트별로 리뷰 의미가 달라지는 일을 줄입니다.

API 키와 코드는 어디에 저장되나요?

CodeAgora는 별도 SaaS 서버에 코드를 쌓아두는 구조가 아닙니다. 사용자가 지정한 모델 공급자와 직접 통신하고, provider API 키는 로컬 파일 권한 경계 안에서 관리하며, 로그와 산출물에서는 비밀값을 가립니다.

Desktop은 안정판인가요?

현재 Desktop은 RC 경로입니다. 로컬 UI로 세션과 리뷰 결과를 다루는 방향이며, 이 랜딩 페이지는 안정판 Desktop 배포나 안정판 업데이트 채널을 약속하지 않습니다.
